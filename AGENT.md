# AGENT.md — „Колелото на живота" (Wheel of Life)

> Интерактивно едностранично уеб приложение за самооценка в 7 сфери на живота.
> Базирано на работния лист на Юли Тонкин / МАБИ (`tonkin.bg`).
>
> **Този файл е единственият източник на истина за проекта. Следвай го стриктно.**

---

## 1. Мисия на проекта

Изгради **едностраничен (single-page) уеб сайт**, който възпроизвежда дигитално и интерактивно упражнението „Колелото на живота – самооценка в 7 сфери".

Потребителят преминава през **3 последователни стъпки** + финален резултат:

1. **СТЪПКА 1 — Оценяване:** попълва оценки 0–10 за 7 сфери чрез слайдери.
2. **СТЪПКА 2 — Изчертаване на колелото (автоматично):** приложението генерира кръг, разделен на 7 равни сектора. Всеки сектор има скала: **0 в центъра → 10 в най-външния край**. Този етап е изцяло автоматичен — потребителят не прави нищо.
3. **СТЪПКА 3 — Поставяне на точки:** потребителят кликва по оста на всеки сектор, за да постави/коригира точка. Точките са предварително позиционирани спрямо слайдерите от Стъпка 1.
4. **РЕЗУЛТАТ — „Изчисли":** при натискане на бутона всички точки се свързват автоматично в плавно очертан полигон (формата на колелото). Под колелото се появява модерен панел с рефлексивните въпроси.

## 4. Детайлна спецификация по стъпки

### СТЪПКА 1 — Оценяване (въпроси)
- Заглавие: „Оцени всяка сфера от 0 до 10"
- Подзаглавие (от PDF): „0 = незадоволително, 10 = оптимално"
- За всяка от 7-те сфери: етикет + **слайдер (range 0–10, стъпка 1)** + текущата стойност се показва до слайдера.
- Всеки слайдер има цвят, съответстващ на своя сектор в колелото (виж §6).
- Долу бутон **„Напред →"**. Активен винаги (слайдерите имат default стойност, напр. 5).
- Плавен вход/изход на секцията с Framer Motion.

### СТЪПКА 2 — Автоматично изчертаване на колелото
- Спусъкът е натискане на „Напред →" от Стъпка 1.
- Приложението **анимирано изрисува**:
  - Външен кръг.
  - 7 радиални линии (оси), които делят кръга на **7 равни сектора** (360° / 7 ≈ 51.43° всеки).
  - Концентрични водещи кръгове за скалата (по желание — фини, за визуален ориентир).
  - Върху всяка ос: числото **0 близо до центъра** и числото **10 в най-външния край**. (Автоматично, потребителят не въвежда нищо.)
  - Етикет на всеки сектор (името на сферата) от външната страна.
- Анимация: осите „израстват" от центъра навън (Framer Motion / SVG stroke-dashoffset).
- Кратка пауза/бутон „Продължи →" към Стъпка 3.

### СТЪПКА 3 — Поставяне на точки
- Колелото вече е изчертано.
- На всяка ос точката е **предварително позиционирана** според слайдера от Стъпка 1 (`point = score`).
- Потребителят може да **кликне върху позиция на всяка ос**, за да премести точката на тази оценка (0–10). Кликът се snap-ва към най-близкото цяло число 0–10.
- Точката е ясно видима (кръгче с цвета на сектора), с hover/active състояние.
- Инструкция на екрана: „Кликни върху всяка ос, за да поставиш или коригираш точката си."
- Долу бутон **„Изчисли ▸"**.

### РЕЗУЛТАТ — Свързване + рефлексия
- При „Изчисли":
  - Всички 7 точки се **свързват в затворен полигон** (формата на колелото).
  - Полигонът се изрисува с плавна анимация (stroke draw + fill с прозрачност).
  - Полупрозрачен цветен fill вътре в полигона.
- Под колелото се появява **модерен панел (card)** с плавна поява отдолу, съдържащ рефлексивните въпроси (точен текст от PDF):
  1. „Помисли как изглежда твоето колело. Би ли се търкаляло плавно?"
  2. „Кое е първото нещо, което искаш да подобриш тази седмица?"
- Втори въпрос има **текстово поле**, където потребителят може да запише отговора си (само локално, без backend).
- Бутон „Начертай отново" (reset към Стъпка 1) + бутон „Изтегли като изображение" (export на SVG/PNG — по желание, nice-to-have).

## 5. Геометрия на колелото (референтна имплементация)

```typescript
const CENTER = { x: 300, y: 300 }; // за viewBox 600x600
const MAX_RADIUS = 240;            // радиус при оценка 10
const N = 7;                       // брой сектори

// Ъгъл на i-тата ос (започваме от -90° = върха, по часовниковата стрелка)
function axisAngle(i: number): number {
  return (-90 + (360 / N) * i) * (Math.PI / 180);
}

// Позиция на точка при оценка value (0..10) по ос i
function pointOnAxis(i: number, value: number) {
  const r = (value / 10) * MAX_RADIUS;
  const a = axisAngle(i);
  return {
    x: CENTER.x + r * Math.cos(a),
    y: CENTER.y + r * Math.sin(a),
  };
}

// Границите на сектор i са между axisAngle(i) - halfStep и + halfStep,
// ако центрираме оста в средата на сектора. Избери един подход и го спазвай
// консистентно за оси, етикети, точки и клик-детекция.
```

**Важно:** осите, етикетите, точките и клик-зоните трябва да използват **една и съща** ъглова конвенция. Дефинирай я веднъж и я преизползвай навсякъде.

## 8. State machine (App.tsx)

```
step: 1 → 2 → 3 → 'result'
```
- `1 → 2`: бутон „Напред →". При прехода: `point = score` за всяка сфера.
- `2 → 3`: бутон „Продължи →" (или auto след анимацията).
- `3 → result`: бутон „Изчисли ▸". Тригерира свързването на точките.
- `result → 1`: бутон „Начертай отново" (reset).

Един споделен `spheres` масив в state; `WheelSvg` е презентационен и приема props (сфери, режим: рисуване/интерактивен/резултат).

## 10. Работен процес за агента

1. Скелетирай проекта (Vite + React + TS + Tailwind + Framer Motion + lucide-react).
2. Настрой `tailwind.config.js` с брандовите токени и цветовете на секторите.
3. Създай `data/spheres.ts` и `lib/geometry.ts` (чисти функции — тествай мислено геометрията).
4. Изгради `WheelSvg.tsx` — прост режим „draw" първо (само оси + скала), провери визуално.
5. Добави слайдерите (Стъпка 1), после клик-точките (Стъпка 3), после полигона (резултат).
6. Свържи всичко в `App.tsx` state machine.
7. Полирай дизайн + анимации + responsive.
8. Провери всеки чекбокс от §9.

**При всяка стъпка:** прави малки, проверими промени. Не пиши целия проект наведнъж — build-вай инкрементално и проверявай, че компилира.
